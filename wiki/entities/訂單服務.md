# 訂單服務

> **【範例】** 服務名稱、URL、團隊均為虛構；僅示範 `entity` 頁應如何盤點系統與引用來源。

## Summary

**訂單服務**（order-service）為交易平台後端團隊維護的訂單領域服務，負責建立、查詢、列表與取消訂單；對外經 API Gateway 暴露 **REST API**（`/v1`）。（確定）[[sources/訂單-api-簡介]]

## Key Points

| 項目 | 說明 |
|------|------|
| 擁有者 | 交易平台後端（虛構） |
| 對外 Base URL | `https://api.example.internal/v1`（範例） |
| 認證 | Bearer JWT（SSO 發行） |
| 文件版本 | 內部簡介稿 v1.3；完整規格指向「訂單服務 API v2」（虛構） |

**主要端點**（確定）[[sources/訂單-api-簡介]]：

| 方法 | 路徑 | 用途 |
|------|------|------|
| POST | `/orders` | 建立訂單（201，支援 `idempotency_key`） |
| GET | `/orders/{order_id}` | 查詢單筆 |
| GET | `/orders` | 列表（`status`、`created_after`） |
| POST | `/orders/{order_id}/cancel` | 取消（僅 `PENDING`） |

**營運**（確定）[[sources/訂單-api-簡介]]：

- 速率限制：600 req/min／API Key；429 + `Retry-After`
- 錯誤追蹤：回應含 `request_id` 供客服／維運對照
